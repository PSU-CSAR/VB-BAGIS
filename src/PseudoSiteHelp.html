Step 1: Find Site finds a new site with the constraints you specify.

Step 2: Add New Site to Scenario 1 and Recalculate adds the new site to scenario 1 and recalculates scenario 1’s represented area.

Step 3: Define a new site with the same settings finds an additional new site using the same constraints. Repeat Step 2 after Step 3 completes to update scenario 1's represented area. Keep repeating Steps 2 and 3 until you have the desired number of sites.

Changing the constraints after Step 1 allows you to ignore the new site and find another site (the Step 1 button will be reactivated).

Editing the constraints after Step 2 allows you to find a subsequent site with different search settings.

The last site created is saved in ArcMap’s memory so you can close the Pseudo Site dialog window to view the Scenario/Site Representation maps.

To update the represented areas captured by the new site, you must add the last site to Scenario 1 (by selecting the checkbox next to the site in the Scenario 1 grid on the Site Scenario dialog window), and recalculate the represented areas (by clicking the Calculate button on the Site Scenario dialog window).

Selecting the log for any auto site will enable Step 3 and allow you to find another site with the same search settings as the log site. In the same ArcMap session, if you select the log for the site most recently added to the represented area calculation, Step 3 will finish faster because BAGIS will re-use the cached constraint layers. If you terminate ArcMap, these cached layers are deleted and can no longer be reused.
